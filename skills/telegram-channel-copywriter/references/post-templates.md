# Post templates for @algorithmic_planetoid

## Template A — Pain -> Insight -> Action
1. Hook: painful dev scenario in 1 line.
2. Insight: why it happens (2-4 lines).
3. Action: 3-step fix checklist.
4. CTA: ask readers for their variant.

## Template B — Myth -> Counterexample -> Rule
1. Hook: common belief.
2. Counterexample from practice.
3. Rule of thumb (do X, avoid Y).
4. CTA: agree/disagree prompt.

## Template C — Mini case
1. Context (project/task).
2. Decision options (A/B).
3. Chosen option + tradeoff.
4. Result and takeaway.
5. CTA: "what would you choose?"

## Template D — Tool breakdown
1. Hook with outcome (not feature).
2. What problem tool solves.
3. Where it fails / limits.
4. Quick setup or usage snippet.
5. CTA: ask for alternatives.

## Template E — Fast checklist
1. Hook: "before you do X, check this".
2. 5-7 concise bullets.
3. One anti-pattern line.
4. CTA: save/share if useful.

## Hook bank
- "Самая дорогая ошибка в <topic> — это ..."
- "Если у вас <symptom>, проблема почти всегда в ..."
- "Контринтуитивно: чтобы ускорить <goal>, сначала замедлите ..."
- "Коротко: <popular advice> не работает, когда ..."

## CTA bank
- "Как у вас это решено в проде?"
- "Нужен разбор с кодом/схемой в следующем посте?"
- "Согласны или это переоценено?"
- "Кинуть шаблон/чеклист в следующем посте?"
